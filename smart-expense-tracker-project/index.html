What Project Are We Building?

Goal: Expense management dashboard banana jo browser mein data save kare.

Is project mein hum Smart Expense Tracker banayenge. User expense title, amount, and category enter karega. JavaScript input validate karega. Valid expense list mein add hoga. Dashboard total spent, total items, highest expense, category filter, and delete button show karega.

Story type se samjho: Ek student ya office employee apne daily expenses track karna chahta hai. Usne lunch, travel, shopping, course fee jaise expenses add kiye. Website automatically total calculate karegi, highest expense batayegi, category filter karegi, aur browser close karne ke baad bhi data localStorage mein save rahega.

FeatureUseJavaScript Concept
Add ExpenseForm se new record add karna.Form submit event, value read.
Total SpentAll expense amount ka sum.Array reduce.
Category FilterFood/Travel/Shopping wise records show karna.Array filter.
Delete ExpenseSpecific record remove karna.Event delegation, dataset id.
Browser SaveRefresh ke baad data keep karna.localStorage, JSON.

What New Things Will Learners Learn?

Goal: Expense dashboard ke through real data handling logic samajhna.

Is project mein learners sirf button click nahi seekhenge. Yahan woh complete data flow seekhenge: form se data lena, object banana, array mein store karna, localStorage mein save karna, filter apply karna, cards render karna, and delete action handle karna.

HTML Line-by-Line Explanation

Is table ko class mein code ke saath-saath read karwao. Story simple hai: HTML ek shop/dashboard ka structure banata hai. Jaise shop mein counter, form, shelves, and display board hote hain, waise hi yahan HTML form, total card, filter, and expense list ke boxes banata hai. JavaScript baad mein in boxes ko id se pakadkar data update karega.

Line / CodeDetailed Simple Hinglish Explanation
<!DOCTYPE html>Yeh first line browser ko clear instruction deti hai ki document modern HTML5 rules follow karega. Agar yeh line missing ho, browser old compatibility mode mein page render kar sakta hai, jisse spacing ya layout weird lag sakta hai. Class mein isko “browser ko rulebook batana” bolkar samjhao.
<html lang="en">Yahan se complete HTML document start hota hai. lang="en" browser, screen reader, and search engine ko batata hai ki page ka main language English hai. Agar future mein Hindi page ho toh lang="hi" use kar sakte hain. Accessibility ke liye yeh small line important hai.
<head>Head section page ka control room hota hai. Iske andar woh information hoti hai jo mostly page par directly visible nahi hoti, jaise title, mobile setting, CSS file connection. Story: restaurant ka kitchen visible nahi hota, but food prepare wahi hota hai. Waise hi browser page settings yahan se read karta hai.
<meta charset="UTF-8">Yeh line text encoding set karti hai. UTF-8 ka matlab page normal letters, numbers, currency symbols, special characters, and many languages ko properly display kar sakta hai. Agar encoding wrong ho, toh symbols ya special text broken boxes jaisa show ho sakta hai.
<meta name="viewport"...>Mobile responsive design ke liye yeh line compulsory samjho. width=device-width browser ko bolta hai ki page ki width device screen ke equal rakho. initial-scale=1.0 page ko default zoom par open karta hai. Iske bina mobile par page zoomed-out ya chhota dikh sakta hai.
<title>Browser tab mein jo naam dikhta hai, woh title tag se aata hai. Is project mein title “Smart Expense Tracker” hai, isliye user ko tab dekhkar pata chalega kaunsi website open hai. SEO and browser history ke liye bhi title useful hota hai.
<link rel="stylesheet"...>Yeh HTML file ko external CSS file se connect karta hai. rel="stylesheet" batata hai ki linked file design rules wali stylesheet hai. href="styles.css" CSS file ka path hai. Agar filename galat hua, HTML structure dikhega but design apply nahi hoga.
<main class="app">main page ka primary content area hai. class="app" CSS ko ek hook deta hai jisse complete app ko center, max width, and padding diya ja sake. Real website mein one page ka main content semantic tag mein rakhna best practice hai.
<section class="hero">Hero page ka top introduction block hai. User sabse pehle isi area ko dekhta hai, isliye yahan project title, short description, and total spent card rakha gaya. class="hero" se CSS is section ko two-column grid banayega.
<p class="eyebrow">Eyebrow small label hota hai jo heading ke upar context deta hai. Jaise “JavaScript Practical Project” user ko immediately batata hai ki yeh project kis topic se related hai. CSS isko uppercase, blue color, and bold style dega.
<h1>Page ka sabse important heading. Is page ka purpose “Smart Expense Tracker” hai, isliye h1 mein wahi likha gaya. One page par normally ek clear h1 hona chahiye, taaki learner, screen reader, and search engine ko main topic samajh aaye.
<strong id="totalAmount">Yeh total spent amount show karne ki jagah hai. Initially Rs. 0 dikhega. JavaScript id="totalAmount" se is exact element ko pakadkar total amount update karega. Id unique hoti hai, isliye JS ko confusion nahi hota.
<form id="expenseForm">Form user input collect karne ka box hai. User title, amount, category fill karega aur Add Expense click karega. id="expenseForm" JavaScript ke liye important hai, kyunki submit event isi form par attach hota hai.
<label for="expenseTitle">Label input ka naam batata hai. for="expenseTitle" label ko us input ke id="expenseTitle" se connect karta hai. Label click karne par input focus ho sakta hai, aur screen reader users ko field ka purpose samajh aata hai.
<input id="expenseTitle"...>Yeh text input expense ka naam lene ke liye hai, jaise Lunch, Bus Ticket, Book, Course Fee. id JavaScript ko value read karne mein help karta hai. placeholder learner ko example hint deta hai.
<input id="expenseAmount" type="number">Yeh amount field hai. type="number" browser ko batata hai ki yahan numeric value expected hai. Mobile par number keyboard open ho sakta hai. JavaScript baad mein is value ko Number() se actual number mein convert karega.
<select id="expenseCategory">Dropdown category choose karne ke liye hai. Free text ke bajay dropdown use karne se data clean rehta hai, jaise Food, Travel, Shopping, Learning. Agar user spelling alag-alag likhe, filtering difficult ho sakti hai; dropdown se same values milti hain.
<option value="">Yeh default empty option hai: “Choose category”. Iska value="" blank hai, isliye JavaScript easily check kar sakta hai ki user ne category choose nahi ki. Validation ke liye yeh line useful hai.
<button type="submit">Button form submit karta hai. type="submit" ka matlab click hone par form ka submit event fire hoga. JavaScript us submit event ko catch karke page reload stop karega aur expense add karega.
<p id="message">Yeh user feedback area hai. Agar input galat hai toh error message show hoga, aur valid expense add hone par success message. JavaScript is paragraph ka text and class dono change karta hai, jisse red/green styling apply hoti hai.
<select id="filterCategory">Yeh second dropdown list filter karne ke liye hai. Add form wali category input data create karti hai, aur filter dropdown existing data ko category ke basis par show/hide karta hai. JavaScript change event par list re-render karta hai.
<strong id="totalItems">Yahan total records count show hota hai. Agar 3 expenses add hue, JavaScript expenses.length se count nikal kar is element mein update karega. Dashboard summary ke liye useful hai.
<strong id="highestExpense">Yeh highest expense amount show karta hai. JavaScript all expense amounts mein se maximum value find karta hai aur yahan display karta hai. Real use case: user ko pata chal jata hai sabse bada kharcha kis range ka hua.
<div id="expenseList">Yeh empty container hai jahan JavaScript expense cards dynamically add karega. HTML mein cards manually nahi likhe, kyunki records user input se banenge. Isliye JS appendChild se cards isi div mein insert karega.
<script src="script.js">Yeh JavaScript file connect karta hai. Body ke end mein script rakhne ka benefit hai ki pehle HTML elements load ho jaate hain, phir JS unko select karta hai. Agar script head mein bina defer ke rakhenge, kabhi-kabhi JS ko elements null mil sakte hain.

const totalItems = document.getElementById("totalItems");
const highestExpense = document.getElementById("highestExpense");
const message = document.getElementById("message");

let expenses = JSON.parse(localStorage.getItem("expenses")) || [];

function saveExpenses() {
  localStorage.setItem("expenses", JSON.stringify(expenses));
}

function showMessage(text, type) {
  message.textContent = text;
  message.className = "message " + type;
}

function getFilteredExpenses() {
  const selectedCategory = filterCategory.value;

  if (selectedCategory === "All") {
    return expenses;
  }

  return expenses.filter(function (expense) {
    return expense.category === selectedCategory;
  });
}

function updateSummary() {
  const total = expenses.reduce(function (sum, expense) {
    return sum + expense.amount;
  }, 0);

  const highest = expenses.length === 0 ? 0 : Math.max(...expenses.map(function (expense) {
    return expense.amount;
  }));

  totalAmount.textContent = "Rs. " + total;
  totalItems.textContent = expenses.length;
  highestExpense.textContent = "Rs. " + highest;
}

function renderExpenses() {
  const filteredExpenses = getFilteredExpenses();
  expenseList.innerHTML = "";

  if (filteredExpenses.length === 0) {
    expenseList.innerHTML = '<p class="empty">No expenses found for this category.</p>';
    updateSummary();
    return;
  }

  filteredExpenses.forEach(function (expense) {
    const card = document.createElement("article");
    card.className = "expense-card";
    card.innerHTML = `
      <div>
        <h3>${expense.title}</h3>
        <p class="expense-meta">${expense.category}</p>
      </div>
      <div>
        <div class="amount">Rs. ${expense.amount}</div>
        <button class="delete-btn" data-id="${expense.id}">Delete</button>
      </div>
    `;
    expenseList.appendChild(card);
  });

  updateSummary();
}

expenseForm.addEventListener("submit", function (event) {
  event.preventDefault();

  const titleValue = expenseTitle.value.trim();
  const amountValue = Number(expenseAmount.value);
  const categoryValue = expenseCategory.value;

  if (titleValue === "") {
    showMessage("Please enter expense title.", "error");
    return;
  }

  if (amountValue <= 0) {
    showMessage("Please enter a valid amount.", "error");
    return;
  }

  if (categoryValue === "") {
    showMessage("Please choose a category.", "error");
    return;
  }

  const expense = {
    id: Date.now(),
    title: titleValue,
    amount: amountValue,
    category: categoryValue
  };

  expenses.push(expense);
  saveExpenses();
  renderExpenses();
  showMessage("Expense added successfully.", "success");
  expenseForm.reset();
});

expenseList.addEventListener("click", function (event) {
  if (event.target.classList.contains("delete-btn")) {
    const expenseId = Number(event.target.dataset.id);
    expenses = expenses.filter(function (expense) {
      return expense.id !== expenseId;
    });
    saveExpenses();
    renderExpenses();
    showMessage("Expense deleted successfully.", "success");
  }
});

filterCategory.addEventListener("change", function () {
  renderExpenses();
});

renderExpenses();
